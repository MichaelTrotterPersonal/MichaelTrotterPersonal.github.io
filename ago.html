Project Background

AGO is one of ten agencies that comprise the National Intelligence Community (NIC), and one of two within the Australian Department of Defence. AGO is the lead in providing geospatial intelligence to Defence and the NIC.

AGO's DIAD is tasked with providing support to analytical teams through the development of technical solutions and methodologies, automating work processes, and aiding in the analysis of geospatial data and imagery. Typically, this is in the form of geoprocessing toolboxes, python packages, python tools, and other machine analytics such as computer vision models and microservices.

This internship project was for the development of a logging solution for DIAD tools and services. The intended outcome was to provide DIAD and senior decision makers in AGO with better business intelligence and improve support to DIAD customers through real-time monitoring of tools.

Better business intelligence
Through the capture of information relating to end-user demographics as well as patterns (or absence) of tool usage, the DIAD and AGO executive are able to make informed decisions in the allocation of human, financial, and physical resources. Moving forward, this will improve the prioritisation of DIAD tasking and ensure that the tools being developed are delivering value to the organisation. Ultimately this will assist AGO in meeting its strategic objectives through the alignment of DIAD work with identified priorities.

Improved user support
With the addition of logging, shortcomings in the design of DIAD tools are now able to be identified in real-time through the capture of unhandled or common errors, runtime information, resource usage, and user tool interactions. This will improve user experience outcomes through timely support and continuous deployment, as well as ongoing user-driven optimisation and improvements to tools. This significantly reduces the burden on users to document and report many commonly identified issues, passively improving user experience outcomes.

Logging Service Overview

This project included the development of a Python logging package, the coordination of infrastructure provisioning, the development of supporting analytics, and creation of documentation for technical and non-technical users.

Logs are constructed and written to corporate storage using the developed ago-logging Python package which is integrated into tools and services. Analytics are delivered via a hosted Streamlit app that is accessed via browser. The Streamlit app allows users to query logs, receive alerts, and view analytics.

Technical and non-technical documentation for both the ago-logging package and the Streamlit app are hosted in various corporate systems such as Gitlab and Confluence.

The package has been included in developer project templates making the logging of future tools largely passive. As logs accumulate over time, the value of the analytics that utilise them will grow.

The developed Streamlit dashboard and analytics are highly extensible and able to adapt to changing organizational requirements.

ago-logging Python Package

In order to capture the required tool and user information, construct the logs, and send them to server-hosted storage, an ago-logging Python package was developed.

This package builds upon the Python Standard Library logging module distributed with Python. It abstracts some of the DIAD business logic, reducing the burden on DIAD developers to integrate logging with their tools and capture the required information at runtime.

The ago-logging package is hosted in a DIAD Gitlab space. CI/CD pipelines were established to automatically build, test, and deploy ago-logging to a Sonatype Nexus repository manager from where it can be pip installed by developers and end-users.

Included in ago-logging is a log_decorator function. This is intended to be the primary way developers integrate logging into their Python tools and services. The minimal usage shown below is sufficient to capture the majority of required information on tool usage, user details, runtime, etc. The decorator also accepts keyword arguments allowing developers to redirect log destinations, set logging levels and custom messages.

The log_decorator instantiates an AGOLogger() class which is also available to developers and offers finer control over logging. As well as capturing many of the details available through the decorator, the functionality of the Python Standard Library logging module is available, including logging at default or custom levels.

Analytics

Shown here is the Log Jam web app developed with Streamlit, an open-source Python library.

Log Jam provides the functionality for filtering and visualising log records. The initial deployment of Log Jam includes a stacked bar chart with user and tool breakdown, a date-usage trendline, and an hour of the week usage heatmap.

Development of analytics is done using common Python packages such as pandas, matplotlib, and seaborn, making the dashboard readily customisable. Future work will include user authentication, allowing for user-specific customisation of analytics.
